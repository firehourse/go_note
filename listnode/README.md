## listnode

鏈表或者說連結串列應該是計算機裡面最常出現的結構了，幾乎所有的基建都是用鏈表來實現的
在我認知中甚至超越了陣列

用go來寫鏈表的話我覺得比其他物件導向的語言來的更簡單清楚，
其實就是把儲存單元像鍊條一樣串起來，他跟陣列的最大差異就是他是離散的
所以當我們要從鏈表查找就會是O(n)

```go
type listNode struct {
    // 隨便宣告一個值，其實是你主要的結構體
    val int
    // 接著宣告一個指標，指向下一個節點
    next *listNode // 由於下一個節點也會是listNode 所以宣告型態
}
```
看到上面的代碼有沒有覺得非常簡單明瞭
iterator CoT 都是在這樣的結構像運行，他非常的實用
